# Contributing

## Commit messages
Use Conventional Commits:

```
<type>(optional-scope): <subject>

Examples:
feat(argo): add autopilot bootstrap
fix(eks): correct nodepool labels
docs: add ADR for GitOps approach
```

Allowed types: feat, fix, docs, chore, refactor, test, build, ci.

## Pull requests
- Keep changes focused and small.
- Update docs and runbooks if behavior changes.
- Run `make lint` and `make fmt` before pushing.
